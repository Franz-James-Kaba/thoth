# Bug Reporting - Validation Checklist

## Summary
- [ ] Clear, descriptive bug title
- [ ] Bug ID assigned
- [ ] Reporter and date documented
- [ ] Environment specified

## Severity & Priority
- [ ] Severity appropriately assessed
- [ ] Priority reflects business urgency
- [ ] Impact on users considered

## Reproduction Steps
- [ ] Preconditions clearly stated
- [ ] Steps are numbered and specific
- [ ] Test data included
- [ ] Expected vs actual results documented
- [ ] Steps can be followed independently

## Environment Details
- [ ] OS and version specified
- [ ] Browser/app version included
- [ ] Application version/build documented
- [ ] Relevant configuration noted

## Evidence
- [ ] Screenshots attached (if applicable)
- [ ] Logs included (console, server, network)
- [ ] Error messages captured
- [ ] Stack traces provided (if available)

## Impact
- [ ] Affected functionality identified
- [ ] User impact quantified
- [ ] Business impact assessed

## Additional Information
- [ ] Workarounds documented (if any)
- [ ] Related bugs referenced
- [ ] Additional context provided
